# Commit Convention

이 프로젝트는 **Conventional Commits** 규칙을 따릅니다.

## 커밋 메시지 형식

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 필수 요소

- **type**: 커밋의 타입
- **subject**: 커밋의 간단한 설명 (50자 이내 권장)

### 선택 요소

- **scope**: 변경 범위 (컴포넌트, 파일명 등)
- **body**: 상세한 설명
- **footer**: Breaking changes, 이슈 참조 등

## Type 종류

| Type | 설명 | 예시 |
|------|------|------|
| **feat** | 새로운 기능 추가 | `feat(canvas): 드로잉 도구 추가` |
| **fix** | 버그 수정 | `fix(renderer): SVG 내보내기 오류 수정` |
| **docs** | 문서 수정 | `docs(readme): 설치 가이드 업데이트` |
| **style** | 코드 포맷팅, 세미콜론 누락 등 | `style(app): prettier로 코드 포맷팅` |
| **refactor** | 코드 리팩토링 | `refactor(utils): 폰트 변환 로직 단순화` |
| **test** | 테스트 코드 추가/수정 | `test(font): 파서 단위 테스트 추가` |
| **chore** | 빌드, 패키지 매니저 설정 등 | `chore(deps): react 버전 18로 업데이트` |
| **perf** | 성능 개선 | `perf(canvas): 렌더링 성능 최적화` |
| **ci** | CI/CD 관련 변경 | `ci(github): 자동화 테스트 워크플로우 추가` |
| **build** | 빌드 시스템 또는 외부 종속성 변경 | `build(vite): 빌드 설정 업데이트` |

## 커밋 메시지 작성 규칙

1. **type과 scope는 영문으로, subject와 body는 한글로 작성**
2. **subject는 명령형으로 작성** (~추가, ~수정, ~삭제 등)
3. **subject 끝에 마침표(.) 사용하지 않음**
4. **제목과 본문은 빈 줄로 구분**
5. **본문은 "무엇을", "왜" 변경했는지 설명**

## 중요 주의사항

⚠️ **AI 워터마크 제거**

AI 도구(ChatGPT, Claude 등)를 사용하여 커밋 메시지를 작성할 경우, 다음과 같은 워터마크를 **반드시 제거**해야 합니다:

- `🤖 Generated with [Claude Code](https://claude.com/claude-code)`
- `Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>`

## 예시

### 기본 예시

```bash
feat(font): 커스텀 폰트 업로드 기능 추가

fix(canvas): 모바일 기기에서 텍스트 렌더링 버그 수정

docs(contributing): 기여 가이드라인 추가

refactor(renderer): SVG 생성 로직 단순화

test(font-parser): 엣지 케이스 테스트 추가

chore(deps): typescript 5.0으로 업그레이드
```

### 상세한 예시 (본문 포함)

```bash
feat(export): PNG 내보내기 기능 추가

커스텀 해상도 설정으로 폰트를 PNG 이미지로 내보낼 수 있는 기능 추가.
사용자는 이제 SVG와 PNG 내보내기 포맷 중 선택할 수 있음.

Closes #123
```

### Breaking Change 예시

```bash
feat(api): 폰트 데이터 구조 변경

BREAKING CHANGE: 폰트 데이터 구조가 업데이트됨.
기존 포맷은 더 이상 지원되지 않으므로 제공된 마이그레이션 스크립트를 사용하여 마이그레이션 필요.
```
